# Your AI agent is reading your .env. One command fixes it.

**Release:** ownlock 0.2.3 / 0.3.0  
**Hook:** Cross-platform agent safety — not macOS-only.

## The problem

Coding agents run in sandboxes that can still read files on disk. `.cursorignore` and `permissions.deny` help, but they're easy to misconfigure and don't cover every tool path.

Meanwhile your `.env` sits in the repo root in plaintext.

## The fix

```bash
pip install ownlock   # or: uv tool install ownlock
ownlock init
ownlock shield --verify
ownlock guard --install-hook
```

- **`ownlock shield`** writes ignore files, Claude deny rules, and a PreToolUse hook that blocks `.env` / `.ownlock` reads.
- **`ownlock guard`** redacts vault values from agent tool output via PostToolUse.
- **`ownlock run`** injects secrets at runtime without putting the master passphrase in the child environment.
- **MCP** delegates to the CLI subprocess — the server never decrypts the vault in-process.
