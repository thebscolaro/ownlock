# Secrets with an approval gate for coding agents

**Release:** ownlock 0.3.0

## What's new

Per-secret policies let you decide how much friction agents (and humans) face before reading a value:

```bash
ownlock set DATABASE_URL --policy confirm   # prompt every time
ownlock set API_KEY --policy session        # unlock for 30 minutes
ownlock set PUBLIC_KEY --policy open        # default
```

Policies apply to `ownlock get`, `vault()` resolution in `.env`, and MCP `ownlock_request_access`.

## Team sync without Slack DMs

```bash
ownlock share --team
git add .ownlock/team.olbundle
```

Teammates run `ownlock init` and get prompted to import the team bundle.

## Why 0.3.0

Schema and behavior changes (policy column, team bundle flow) deserve a minor bump. Shield/guard shipped in the same release cycle for momentum.
